\chapter{Resultados Parciais e Plano de Trabalho}
\label{cap:Results}

Neste capítulo são apresentados os primeiros experimentos do estudo comparativo proposto no Capítulo \ref{cap:metodology}. O estudo foi aplicado a três contramedidas sendo duas explorando as características relativas a qualidade da imagem e uma relativa às características da cena. Das três contramedidas, uma foi implementada pelo autor. Vale ressaltar que os códigos fonte destas contramedidas estão livremente disponíveis na WEB com instruções de como reproduzir os resultados de cada trabalho de referência. As contramedidas são:
\begin{itemize}
	\item Contramedida que explora a relação de movimento da face em relação a cena proposta por \cite{AnjosIJCB2011}\footnote{http://pypi.python.org/pypi/antispoofing.motion/1.0.2};
	\item Contramedida que explora texturas utilizando LBP proposta por \cite{ChingovskaBIOSIG2012}\footnote{http://pypi.python.org/pypi/antispoofing.lbp/1.1.0};
	\item Contramedida que explora a dinâmica de uma textura ao longo do tempo utilizando $LBP-TOP$\cite{Pereira_LBP_2012}\footnote{http://pypi.python.org/pypi/antispoofing.lbptop/}.
\end{itemize}

As contramedidas propostas por \cite{AnjosIJCB2011} e \cite{Pereira_LBP_2012} admitem apenas vídeos como entrada. Por esta razão, a metodologia foi aplicada apenas nas bases de dados Replay Attack e CASIA FASD que possuem apenas vídeos. Neste experimento, a configuração de cada contramedida foi selecionada seguindo orientações apresentadas em cada trabalho de referência. Mais detalhes sobre as configurações de cada contramedida podem ser obtidas no Capítulo \ref{cap:review}.

Em especial, a contramedida proposta por \cite{AnjosIJCB2011} utiliza redes neurais do tipo MLP para classificação. Dada a natureza aleatória na inicialização dos pesos sinápticos, experimentos com esta contramedida foram executados cinco vezes. A média dos resultados é reportada como resultado final.

A comparação da performance de cada contramedida em diferentes bases de dados sugere que a base de dados CASIA FASD possui ataques mais difíceis de serem detectados que os da base de dados Replay Attack.

Este comportamento pode ser também observado através das curvas ROC na Figura \ref{fig:ROC_cross}. As curvas azuis e verdes (linha pontilhada e linha tracejada) são as performances obtidas no conjunto de calibração de uma base de dados e no conjunto de teste de outra base de dados. É possível observar que as curvas estão bem distantes, ou seja, não é possível ter uma performance satisfatória para qualquer valor de limiar $\tau$. Corroborando com estes resultados, uma análise em termos de FAR 100 e FAR 1000, tem-se um FRR acima de 97\% para todas as contramedidas, impossibilitando o seu uso para aplicações que necessitam de altos requisitos de segurança.

\label{sec:plan}
