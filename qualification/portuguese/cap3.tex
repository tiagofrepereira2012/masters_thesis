\chapter{Metodologia}
\label{cap:metodology}

Neste capítulo serão apresentados as etapas de desenvolvimento deste projeto. Primeiramente é necessário avaliar se as contramedidas são realmente efetivas aplicando um protocolo único a fim de comparar diretamente cada contramedida. Como segunda contribuição um protocolo para avaliação do poder de generalização foi proposto



\section{Protocolo de avaliação Intra-base de dados}
\ref{sec:protocol}

Nesta avaliação pretende-se avaliar a performance das contramedidas, em termos de detecção de ataques, utilizando apenas uma base de dados.

Para comparação escolhemos o Taxa média de erro onde pondera HTER


\begin{equation}
\label{eq:HTER}
HTER=\frac{FAR(\tau,D)+ FRR(\tau,D)} {2} ,
\end{equation}
onde $\tau(D_n)$ é o limiar, $D_n$ é a base de dados, $FAR$ é a taxa de falsas aceitações na base de dados $D_1$ e FRR é a taxa de falsas rejeições na base de dados $D_1$.

\section{Protocolo de avaliação Inter-base de dados}
\ref{sec:protocol}

Nesta avaliação pretende-se avaliar a performance das contramedidas em termos de detecção de ataques em um cenário mais realístico. Este cenário consiste em treinar e calibrar as contramedidas utilizando uma base de dados e reportar os resultados utilizando outras bases de dados. 

\begin{equation}
\label{eq:HTER}
HTER(D_1)=\frac{FAR(\tau(D_2),D_1)+ FRR(\tau(D_2),D_1)} {2} ,
\end{equation}
onde $\tau(D_n)$ é o limiar, $D_n$ é a base de dados, $FAR$ é a taxa de falsas aceitações na base de dados $D_1$ e FRR é a taxa de falsas rejeições na base de dados $D_1$.
